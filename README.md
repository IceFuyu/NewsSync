## Architecture

The site is a single static HTML/CSS/JS file (`index.html`) â€” no build step, no framework, no external dependencies. It makes **zero client-side API calls**.

Instead, [`scripts/generate-news.js`](scripts/generate-news.js) runs server-side via a **manually triggered** GitHub Actions workflow (see [`.github/workflows/update-news.yml`](.github/workflows/update-news.yml)). For each entry in the `TOPICS` config (currently `ki` and `finanzen`) it:

1. **Fetches that topic's RSS feeds directly** â€” free, no dependencies, ~60â€“70 articles per topic.
2. **Filters in code** â€” drops duplicate URLs, anything already in `archive.json`, and anything older than `MAX_AGE_DAYS` (guards against feeds that quietly stop updating). Caps each source at `PER_SOURCE` items so a prolific publisher can't crowd out the rest.
3. **Makes one Claude call per topic** (`claude-haiku-4-5`, no web search) that picks the most important items and writes the German summary, category, tags, and deep-dive in a single response.
4. **Re-attaches source, URL, and image from the real RSS data** â€” the model only returns the index of each item it picked, so links and sources cannot be hallucinated.

A separate small call per topic generates `weekly-review.json` from archive data. The page fetches the resulting `news.json`, `archive.json`, and `weekly-review.json` directly. This keeps the API key server-side only, needs no backend or database, and runs anywhere static files can be served â€” currently GitHub Pages.

**Cost:** roughly 5â€“8 cents per run. An earlier version had Claude research the news itself via the web search tool, which cost ~50Ã— more (~$2 per run) because every search injects its results into the context window.

**Testing feeds:** `npm run dry-run` fetches and prints the candidate articles without calling the model â€” no API key needed, no cost. Use it after changing any feed URL.

Adding a topic requires extending the `TOPICS` array (including its `feeds` list) and the matching `Einstieg`/`Nachschlagewerk` content in `index.html`.
