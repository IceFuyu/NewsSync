# Signal — AI Intelligence

Private AI news dashboard in German: a static site with a Twitter-style feed of curated AI news, AI-generated deep-dives, a beginner's guide, and a model directory. Data comes from JSON generated by a GitHub Actions cron (Claude + web search) — no client-side API calls, no backend, no login.

## What it is

Signal is a private, single-purpose dashboard for staying current on AI news without relying on social media. Built for personal use — for myself and friends — not as a public product: no login, no community features, no analytics, no monetization.

**Two core functions:**
- **Feed** — a scrollable, Twitter/X-style stream of the 8 most important AI stories (models & tools, business, research, policy), each with a short German summary. Clicking a post opens a deep-dive: what happened, why it matters, impact, historical context, and a one-line takeaway.
- **Einstieg (Getting Started)** — a static, jargon-free section for complete beginners: foundational cards on what LLMs are and how they work, practical prompting tips and common mistakes, and a 12-term glossary. No API calls, loads instantly, works offline.

Two supporting features round it out: a **14-day archive** with automatic AI-generated weekly recaps, and a **curated directory of 14 AI models** (GPT, Claude, Gemini, Grok, Llama, Mistral, DeepSeek, Qwen, Flux, Midjourney, Veo, Runway, Kling, Suno) with strengths, weaknesses, use cases, and pricing tier.

## Architecture

The site is a single static HTML/CSS/JS file (`index.html`) — no build step, no framework — with a dark, glassmorphic UI and a Three.js particle/wireframe background. It makes **zero client-side API calls**.

Instead, [`scripts/generate-news.js`](scripts/generate-news.js) runs server-side on a GitHub Actions cron schedule (twice daily, see [`.github/workflows/update-news.yml`](.github/workflows/update-news.yml)), calls the Anthropic API with the web search tool to research and summarize the day's AI news, and writes the results to static `news.json`, `archive.json`, and `weekly-review.json` files. The page fetches these directly. This keeps the API key server-side only, needs no backend or database, and runs anywhere static files can be served — currently GitHub Pages.

## Setup

1. Add a repository secret `ANTHROPIC_API_KEY` under **Settings → Secrets and variables → Actions**.
2. Enable **Settings → Actions → General → Workflow permissions → Read and write permissions** (so the cron job can commit updated data back).
3. Enable **Settings → Pages → Source: `main` / `(root)`**.
4. Trigger the workflow once manually (**Actions → Update News → Run workflow**) to generate the first `news.json`.

## Stack

Vanilla HTML/CSS/JS · Three.js (CDN) · Node.js (data-generation script) · GitHub Actions (cron + secrets) · GitHub Pages (hosting) · Anthropic API (Claude + web search) for news research, deep-dives, and weekly recaps.
