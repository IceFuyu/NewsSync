# NewSync

Private news dashboard in German, styled like an X/Twitter timeline: a shared, filterable feed covering **AI** and **Finance**, AI-generated deep-dives, a beginner's guide, and a curated reference directory per topic. Data comes from JSON generated by a GitHub Actions cron (Claude + web search) — no client-side API calls, no backend, no login.

## What it is

NewSync is a private, single-purpose dashboard for staying current on two topics without relying on social media. Built for personal use — for myself and friends — not as a public product: no login, no community features, no analytics, no monetization.

**Layout:** an X-style three-column app shell — left sidebar for navigation and a topic switcher (KI / Finanzen), a center timeline, and a right "most mentioned this week" sidebar computed from real archive data (no fabricated engagement numbers). Collapses to an icon-only sidebar on tablet and a bottom tab bar on mobile.

**Core functions:**
- **Feed** — a continuous, X-style timeline combining both topics, filterable by topic ("Alle" / "KI" / "Finanzen"). Each post has a short German summary; clicking it opens a deep-dive: what happened, why it matters, impact, historical context, and a one-line takeaway.
- **Einstieg (Getting Started)** — a static, jargon-free section for complete beginners, separate per topic: foundational cards (what LLMs are / what investing is, etc.), practical first-step guides, and a 12-term glossary. No API calls, loads instantly, works offline.
- **Nachschlagewerk (Reference)** — a curated directory, separate per topic: 14 AI models (GPT, Claude, Gemini, Grok, Llama, Mistral, DeepSeek, Qwen, Flux, Midjourney, Veo, Runway, Kling, Suno) with strengths/weaknesses/use cases/pricing tier, or 10 investment asset classes (stocks, ETFs, bonds, crypto, gold, REITs, etc.) with strengths/weaknesses/use cases/risk level.
- **Archiv** — a 14-day archive of both topics combined, with an on-demand weekly recap per topic (AI-generated from archive data only, no web search).

## Architecture

The site is a single static HTML/CSS/JS file (`index.html`) — no build step, no framework, no external dependencies. It makes **zero client-side API calls**.

Instead, [`scripts/generate-news.js`](scripts/generate-news.js) runs server-side on a GitHub Actions cron schedule (twice daily, see [`.github/workflows/update-news.yml`](.github/workflows/update-news.yml)). It loops over a `TOPICS` config (currently `ki` and `finanzen`), calls the Anthropic API with the web search tool to research and summarize each topic's news, tags every item with its `topic`, and writes the combined results to static `news.json`, `archive.json`, and a per-topic `weekly-review.json`. The page fetches these directly. This keeps the API key server-side only, needs no backend or database, and runs anywhere static files can be served — currently GitHub Pages. Adding a third topic later only requires extending the `TOPICS` array and the matching `Einstieg`/`Nachschlagewerk` content in `index.html`.

## Setup

1. Add a repository secret `ANTHROPIC_API_KEY` under **Settings → Secrets and variables → Actions**.
2. Enable **Settings → Actions → General → Workflow permissions → Read and write permissions** (so the cron job can commit updated data back).
3. Enable **Settings → Pages → Source: `main` / `(root)`**.
4. Trigger the workflow once manually (**Actions → Update News → Run workflow**) to generate the first `news.json`.

## Stack

Vanilla HTML/CSS/JS · Node.js (data-generation script) · GitHub Actions (cron + secrets) · GitHub Pages (hosting) · Anthropic API (Claude + web search) for news research, deep-dives, and weekly recaps.
